## 一、v3.0 设计理念

### 1.1 设计愿景

NEXUS v3.0 控制台定位为**轻量化 SaaS 风格的现代数据资产管理平台**，面向中小型组织（教研团队、院校、中型企业），强调"开箱即用、可视化驱动、零学习曲线"。

### 1.2 六大设计原则

| 原则 | 说明 | 设计表现 |
|------|------|----------|
| **简洁高效** | 一屏一事，核心操作 ≤ 3 步完成 | 减少嵌套弹窗，优先内联编辑和侧边抽屉 |
| **可视化驱动** | 数据状态一目了然，决策有据可依 | 仪表盘、趋势图、状态卡片、流程可视化 |
| **模块化** | 功能独立、按需组合、可扩展 | 组件化设计系统，页面即模块拼装 |
| **云原生美学** | 现代、轻盈、有呼吸感 | 柔和渐变、微阴影、圆角卡片、充足留白 |
| **色彩舒适** | 降低视觉疲劳，强调信息层级 | 克制的主色 + 功能色体系，低饱和度中性色 |
| **响应式友好** | 桌面端优先，兼顾平板 | 弹性栅格、自适应卡片、折叠导航 |

### 1.3 设计语言对比

| 维度 | v2.2 原型 | v3.0 原型 |
|------|-----------|-----------|
| 布局风格 | 传统后台管理面板 | 现代 SaaS 卡片流 |
| 导航模式 | 左侧全高固定导航 | 左侧收起式图标导航 + 展开面板 |
| 色彩体系 | 功能色分离 | 品牌主色 + 语义色 + 中性色阶梯 |
| 工作台 | 表格 + 统计卡片 | 可视化仪表盘 + 趋势图 + 快捷操作 |
| 详情页 | Tab 切换 | 分区卡片 + 侧边导航锚点 |
| 状态表达 | 文字标签 | 状态点 + 语义色徽章 + 动画过渡 |
| 空态 | 文字提示 | 插画 + 引导文案 + CTA 按钮 |
| 移动端 | 不考虑 | 响应式折叠至单列 |

### 2.2 字体与排版

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| H1 | 28px / 1.3 | 700 | 页面标题 |

### 2.3 圆角与阴影

```
圆角系统：
  xs: 4px    (标签、徽章、小按钮)
  sm: 8px    (输入框、下拉、按钮)
  md: 12px   (卡片、面板)
  lg: 16px   (弹窗、抽屉)
  xl: 24px   (大卡片、图表容器)

阴影系统：
  none:  none
  sm:    0 1px 2px rgba(0,0,0,0.04)
  md:    0 4px 12px rgba(0,0,0,0.06)
  lg:    0 8px 24px rgba(0,0,0,0.08)
  xl:    0 16px 48px rgba(0,0,0,0.10)
```

### 2.4 间距系统

基于 4px 基础单位：

| Token | 值 | 用途 |
|-------|-----|------|
| xs | 4px | 紧凑间距 |
| sm | 8px | 组件内间距 |
| md | 16px | 区块内间距 |
| lg | 24px | 卡片间距、页面边距 |
| xl | 32px | 大区块间距 |
| 2xl | 48px | 页面顶部底部留白 |

### 2.5 图标系统

使用 **Lucide Icons** 线性图标集，统一 20px 尺寸：
- 导航图标：20px, stroke-width 2
- 操作图标：16px, stroke-width 2
- 状态图标：12px, stroke-width 2.5

### 2.6 动效规范

| 类型 | 时长 | 缓动 | 场景 |
|------|------|------|------|
| Instant | 0ms | — | 状态切换、Tab 切换 |
| Micro | 150ms | ease-out | 悬停、聚焦、按钮反馈 |
| Standard | 250ms | ease-in-out | 抽屉/弹窗进出、展开/收起 |
| Flow | 400ms | ease-in-out | 页面过渡、路由切换 |
| Loading | 800-1200ms | ease-in-out | 骨架屏闪烁、进度条 |

## 二十一、端到端流程 v3.0

### 流程 A：文档入库到自动可用（v3.0 路径）

```
工作台 [快捷操作：上传文档]
  → 数据接入（自动选择默认数据源，拖拽上传）
  → 提交后自动跳转作业中心
  → 作业详情流水线可视化：●→●→●→◉→○→○→○
  → 全部完成后 → 治理中心 AI 建议自动通过
  → 资产目录卡片显示 ●当前可用 + 质量分 92
  → 点击进入资产详情，查看完整治理链路
```

### 流程 B：人工复核（v3.0 路径）

```
工作台 [待处理事项：需人工复核 23]
  → 治理中心 [待复核 Tab]
  → 打开复核抽屉
  → 左侧 AI 建议 + 质量雷达图，右侧规则冲突
  → 裁定分级/标签/组织范围
  → 确认复核通过
  → Toast 通知：复核通过，资产已进入可检索索引
```

### 流程 C：重处理（v3.0 路径）

```
作业中心 [失败作业]
  → 打开作业抽屉，查看失败阶段 ✕
  → 点击 [重处理]
  → 确认弹窗：选择重处理范围（重解析/重标准化/重建索引）
  → 新作业创建，流水线重新开始
  → 新旧版本关系在资产详情版本历史中可追溯
```
