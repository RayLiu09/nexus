# 企业数据与知识资产平台需求 Spec v1.0 — NEXUS

## 一、文档说明

本文档基于 [企业数据与知识资产平台nexus_v7.0](/home/bjbodao/projects/MinerU/企业数据与知识资产平台nexus_v7.0.md) 输出平台需求规格说明，定义一期建设范围、角色、功能需求、非功能需求与验收口径。

本文档用于：

1. 统一业务、产品、研发、测试、实施对平台范围的理解。
2. 作为研发实施、测试验收和交付评审的依据。
3. 为后续版本迭代提供需求基线。

## 二、项目背景与建设目标

### 2.1 背景问题

当前企业内外部非结构化数据存在以下共性问题：

1. 数据分散在网盘、NAS、系统库表、爬虫结果和业务文件中，缺少统一入口。
2. 相同文档在不同项目中重复处理，缺少原始留存和标准化资产沉淀。
3. 文档解析结果、标签规则、检索索引和权限控制之间没有统一治理模型。
4. 上层业务系统和 AI 应用需要重复建设知识库、问答语料和检索服务。

### 2.2 建设目标

平台建设目标如下：

1. 构建统一的数据源接入层、原始数据持久化层和作业编排处理层。
2. 构建统一的标准化资产模型，形成数据资产与知识资产的共同输入件。
3. 建立分类、分级、标签、版本、权限、审计等治理体系。
4. 对外提供统一的资产查询、检索、问答和作业接口。
5. 为 AI 助教、知识检索、教材生成、图谱构建、评价标准化等场景提供底座支撑。

## 三、术语定义

| 术语 | 定义 |
|------|------|
| 数据资产 | 经过接入、登记、留存和治理后，可被统一管理和复用的数据对象 |
| 知识资产 | 在数据资产基础上经过切片、抽取、结构化或审核加工后形成的可直接消费对象 |
| 原始对象 `raw_object` | 原始文件、原始 JSON 包、原始媒体引用等可信留存对象 |
| 文档资产 `document_asset` | 平台侧文档主实体，承载分类、分级、权限和生命周期 |
| 资产版本 `document_version` | 文档资产的版本实例，关联解析产物和标准化产物 |
| 标准化文档 `normalized_document` | 面向 PDF/Word/PPT/图片等文档输入的统一标准对象 |
| 标准化记录 `normalized_record` | 面向 JSON/Excel/数据库同步数据的统一标准对象 |
| 知识切片 `knowledge_chunk` | 用于检索、问答和知识加工的标准切片对象 |
| 索引清单 `index_manifest` | 记录索引分区、同步状态、失败原因与同步时间的清单对象 |
| MinerU | 非结构化文档解析引擎 |
| RAGFlow | 切片、索引与检索执行引擎 |

## 五、范围定义

### 5.1 一期范围

一期建设范围固定如下：

1. 数据源接入：文件上传、目录导入、NAS 同步、爬虫推送。
2. 原始数据持久化：原始对象留存、校验摘要、版本快照、接入台账。
3. 作业编排：持久化作业中心、重试补偿、失败回查、人工复核。
4. 文档解析：基于 MinerU 的 PDF、Word、PPT、图片解析。
5. 标准化处理：生成 `normalized_document`、`normalized_record`。
6. RAGFlow 集成：数据集映射、切片执行、索引构建、检索执行。
7. 治理能力：分类、分级、标签、版本、生命周期、质量复核。
8. 权限审计：RBAC、组织范围、脱敏、审计日志。
9. 服务开放：`nexus-console`、`nexus-api`。
10. 部署交付：单节点与三节点集群方案。

### 5.2 一期数据域范围

| 数据域 | 范围级别 | 说明 |
|------|---------|------|
| D1 产业行业知识域 | 一期纳入 | 静态文档与动态数据均纳入 |
| D2 岗位职业知识域 | 一期纳入 | 招聘数据、职业能力分析、证书、职业大典 |
| D3 专业建设知识域 | 一期纳入 | 教学标准、专业布点、人才培养方案 |
| D4 课程教学知识域 | 一期纳入 | 教材、教案、课件、习题、实训指导书、媒体元数据 |
| D5 教学行为与平台数据域 | 二期预留 | 平台行为数据、用户与课程信息 |
| D6 能力评价与就业数据域 | 二期预留 | 能力评估结果、职业匹配度数据 |

### 5.3 一期不纳入范围

以下内容不在 v1.0 的交付范围内：

1. D5/D6 全量正式接入与实时流处理。
2. 知识图谱完整生产化建设。
3. SFT 训练语料大规模量产。
4. 独立的运维观测中心完整产品化实现。
5. 全链路双活和跨机房容灾架构。

## 六、业务场景

### 6.1 场景一：静态文档入库

业务方上传教材、产业报告、人才培养方案等文档，平台完成原始留存、解析、标准化、索引和资产入库。

### 6.2 场景二：爬虫动态数据接入

爬虫系统按批次推送产业政策、岗位招聘和人才需求数据，平台完成批次登记、原始包留存、标准化处理与索引更新。

### 6.3 场景三：资产治理

数据管理员对新入库资产执行分类分级、标签确认、版本状态管理和人工复核。

### 6.4 场景四：知识检索与问答

上层系统或业务用户通过 `nexus-api` 或控制台执行检索和问答，平台返回带引用、带权限过滤的结果。

### 6.5 场景五：重处理与版本切换

当规则升级、解析失败或人工复核发现问题时，平台触发重处理并生成新版本资产。

### 6.6 场景六：知识资产加工

基于标准化资产和知识切片，平台生成问答语料、流程语料、图谱结构和评价标准库。

## 九、数据要求

### 9.1 支持的数据类型

| 类型 | 范围 |
|------|------|
| 文档类 | PDF、Word、PPT、TXT、Markdown、图片 |

| 半结构化类 | JSON、Webhook 事件 |
| 媒体类 | 视频、动画、应用包、外部链接元数据 |

### 9.2 数据留存要求

1. 原始对象必须留存。
2. 解析产物必须留存。
3. 标准化产物必须留存。
4. 索引状态必须可回查。
5. 任何检索或问答结果必须可回溯至资产版本与原始对象。

### 9.3 元数据要求

平台元数据至少覆盖以下维度：

1. 来源信息
2. 分类信息
3. 分级信息
4. 标签信息
5. 时间信息
6. 质量信息
7. 血缘信息
8. 权限范围信息
9. 索引状态信息

## 十一、验收口径

### 11.1 功能验收

一期验收通过需同时满足以下条件：

1. 文档可统一接入、留存、解析、标准化、索引和检索。
2. 控制台可完成数据源管理、作业查询、标签审核、资产查看和权限管理。
3. API 可完成资产查询、检索、问答、作业查询和重处理触发。
4. 权限与审计链路可运行，L4 数据默认脱敏。
5. RAGFlow 集成链路完整，`index_manifest` 可回查。

## 十二、版本演进说明

### 12.1 v1.0 范围结论

v1.0 的核心定位是：

1. 夯实非结构化数据处理底座。

3. `ops-observability` 独立产品化。
