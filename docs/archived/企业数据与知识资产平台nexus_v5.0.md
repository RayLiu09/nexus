  # 企业数据与知识资产平台nexus_v5.0

  ## 一、项目定位

  NEXUS 面向企业现有非结构化数据，建设统一的数据沉淀、治理、复用与开放平台，解决企业数据分散、处理重复、管理粗放、使用门槛高的问题。

  平台沉淀两类核心资产：

  1. 面向业务应用的数据资产
  2. 面向大模型和智能应用的知识资产

  平台作为企业级数据与知识资产基础设施，统一支撑资产管理和能力开放两类核心场景。

  ## 二、产品结构

  ### 外部系统集成

  数据资产平台集成以下外部系统，形成完整能力闭环：

  1. 实时动态数据源爬虫系统，定期爬取产业政策数据和行业人才需求数据，作为平台实时动态数据源的重要输入通道。
  2. RAGFLOW系统，提供检索、召回、知识组织能力，支撑平台统一的知识服务链路。

  平台采用“一个统一底座 + 两类终端”的产品结构。

  ### 1. 统一数据资产底座

  统一数据资产底座承载平台核心能力，包括以下五项：

  #### 1.1 数据源接入

  平台统一接入静态数据源和实时动态数据源。

  静态数据源包括：

  1. 制度文件
  2. 行业研究资料
  3. 项目交付文档
  4. 历史知识材料
  5. 业务专题文档

  实时动态数据源包括：

  1. 由爬虫系统提供的产业政策数据
  2. 由爬虫系统提供的行业人才需求数据
  3. 其他按周期采集和增量同步的外部动态信息

  #### 1.2 数据解析与清洗

  平台对接入数据执行标准化处理流程，形成可治理、可检索、可复用的数据资产。

  处理环节包括：

  1. 文件内容解析，提取正文、标题、时间、来源、附件等基础内容
  2. 文档结构识别，识别章节、段落、表格、图片说明等结构信息
  3. 格式标准化，统一编码、时间格式、字段格式和文本格式
  4. 内容去噪，去除空白页、页眉页脚、水印、模板语和无效字符
  5. 重复检测，识别重复文档、重复段落和高相似内容
  6. 内容切片，按段落、主题或规则进行切分，为检索和知识组织提供基础单元
  7. 元数据抽取，提取来源、发布时间、所属部门、主题领域、适用范围等元信息
  8. 质量校验，检查解析完整性、字段缺失、格式异常和内容有效性
  9. 清洗结果入库，将结构化结果、切片结果和元数据统一入库沉淀

  #### 1.3 分类分级与标签治理

  平台建立统一的分类分级与标签治理机制，保障资产可组织、可检索、可管理。

  具体内容包括：

  1. 分类体系建设，按业务领域、数据主题、资产类型建立统一分类目录
  2. 分级规则建设，按公开范围、敏感程度、使用对象建立数据分级规则
  3. 标签体系建设，建立主题标签、来源标签、部门标签、时效标签、场景标签等标准标签
  4. 标签规则维护，明确标签命名规则、使用规则和更新规则
  5. 自动标签辅助，对解析后的内容进行基础标签识别与推荐
  6. 人工校正机制，支持管理员对分类、分级和标签结果进行复核与修订
  7. 治理规则沉淀，将分类、分级、标签规则沉淀为统一治理标准

  #### 1.4 权限与可见范围控制

  平台统一控制资产访问权限和结果可见范围，确保不同角色在授权范围内使用数据与知识资产。

  控制内容包括：

  1. 按用户、角色、部门配置访问权限
  2. 按资产分类、数据级别和业务范围控制可见范围
  3. 按终端类型控制管理权限和调用权限
  4. 在检索、查询和接口调用环节执行权限校验
  5. 对访问行为和调用行为保留审计记录

  #### 1.5 检索、召回、知识组织

  平台通过 RAGFLOW 系统提供统一的检索、召回和知识组织能力。

  核心内容包括：

  1. 基于资产切片结果建立统一检索索引
  2. 支持关键词检索、语义检索和混合检索
  3. 支持按分类、标签、来源、时间等条件过滤召回结果
  4. 对召回内容进行聚合、排序和知识组织
  5. 为上层系统提供标准化检索与知识服务能力

  ### 2. `nexus-console`

  `nexus-console` 面向数据管理员，承担资产管理与治理入口职责，覆盖数据源接入管理、解析清洗配置、分类分级与标签治理、权限控制和资产运营管理。

  ### 3. `nexus-api`

  `nexus-api` 面向上层业务系统，以标准化接口开放资产查询、检索召回、知识组织和权限校验能力。

  ## 三、核心价值

  1. 将分散数据沉淀为企业可持续复用的资产，减少重复加工和重复建设。
  2. 为 FLOW 及后续业务系统提供统一的数据与知识底座。
  3. 为上层业务应用提供统一的标准化能力输出接口。
  4. 为知识问答、智能检索、智能助手等 AI 场景提供统一支撑。
  5. 推动数据管理从项目式交付转向平台化、标准化、服务化运营。

  ## 四、两类终端职责

  ### `nexus-console`

  核心职责：

  1. 数据源接入管理
  2. 数据解析与清洗流程配置
  3. 分类分级与标签治理管理
  4. 权限与可见范围控制
  5. 数据处理任务监控与运行管理
  6. 资产运营、统计与审计

  ### `nexus-api`

  核心职责：

  1. 数据资产查询 API
  2. 检索与召回 API
  3. 知识组织结果服务 API
  4. 权限校验 API
  5. 鉴权、限流、审计、监控能力

  ## 六、项目结果

  项目完成后，平台形成以下交付成果：

  1. 统一数据资产底座
  2. 面向数据管理员的 `nexus-console`
  3. 面向业务系统的 `nexus-api`
  4. 首批静态与实时动态数据源接入能力
  5. 分类分级、标签治理、权限控制和检索召回基础能力
