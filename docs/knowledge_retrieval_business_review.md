- **面向读者**：产品、业务、教研、企业方对接人

## 一、平台承载哪些数据资产

平台已汇聚的资产按业务用途分四大族，本质分两大类：

| 资产族     | 具体资产                                 | 资产性质       | 主要业务用途                       |
| ---------- | ---------------------------------------- | -------------- | ---------------------------------- |
| 行业与产业 | 产业政策、产业报告、行业报告             | 知识型（文档） | 了解宏观政策、行业动态、趋势与观点 |
| 教材与课程 | 理论型 / 实训型 / 混合型教材             | 知识型（文档） | 教学备课、学习查阅、任务演练指导   |
| 专业信息   | 专业简介                                 | 知识型（文档） | 专业介绍、课程体系、证书与升学路径 |
| 结构化数据 | 岗位需求数据、职业能力分析表、专业布点数 | 数据型（表格） | 岗位画像、能力对标、专业规划参考   |

> 本阶段业务讨论只覆盖以上已冻结加工方案的资产。其它资产（专业教学标准、人才培养方案、专业人才需求报告、职业技能证书标准）暂不纳入检索增强讨论，后续版本再补齐。

两类资产的检索方式天然不同：

- **知识型资产**：用户提问偏"问答式"，需要按语义查找相关段落，把段落背后的知识结构（政策条款、任务步骤、专业卡片）一起呈现。
- **数据型资产**：用户提问偏"查询式"，需要按字段做筛选和统计（按城市 / 按年份 / 按专业代码），返回记录列表或统计视图。

结论：检索入口必须先识别用户到底想查哪一类，再走对应路径。

## 二、业务用户的典型提问场景

按业务实际，用户提问大致可归为 7 类：

| 场景            | 用户提问举例                                                       |
| --------------- | ------------------------------------------------------------------ |
| 概念 / 定义查询 | "什么是直播电商？"、"跨境电商的定义是什么？"                       |
| 政策 / 规则查询 | "国家对电商企业的融资政策有哪些？"、"直播电商监管条款有哪些？"     |
| 任务 / 操作查询 | "如何完成数据行列转换？"、"转置操作的下一步是什么？"               |
| 专业信息查询    | "电子商务专业核心课程有哪些？"、"跨境电商专业的接续本科有哪些？"   |
| 岗位与薪资查询  | "杭州电商运营岗位的薪资分布？"、"电商运营岗位学历要求分布？"       |
| 能力与任务查询  | "电商运营岗位需要哪些能力？"、"商品运营下有哪些工作任务？"         |
| 专业布点查询    | "近三年高职电子商务专业布点数变化？"、"哪些省份跨境电商专业最多？" |

前 4 类偏"查内容"，后 3 类偏"查数据"。

## 三、平台如何理解用户意图并选择路径

平台在检索入口做两级意图识别。

### 3.1 第一级：判断是"查内容"还是"查数据"

| 用户提问特征                                                                           | 判断                         |
| -------------------------------------------------------------------------------------- | ---------------------------- |
| 出现"什么是"、"如何"、"讲了什么"、"原文位置"                                           | 内容查询（走文档知识路径）   |
| 出现"多少"、"分布"、"排名"、"按…统计"，或字段名（专业代码 / 布点数 / 岗位类别 / 薪资） | 数据查询（走结构化数据路径） |
| 两类信号都出现                                                                         | 双路径并行，结果混合展示     |

### 3.2 第二级：识别业务领域

| 业务领域        | 触发词举例                   | 主要指向的资产           |
| --------------- | ---------------------------- | ------------------------ |
| 知识 / 理论查询 | 概念、定义、政策、报告、原理 | 政策 / 报告 / 理论型教材 |
| 任务 / 操作查询 | 步骤、如何、下一步           | 实训型教材、任务型教材   |
| 专业查询        | 专业代码、职业面向、核心课程 | 专业简介                 |
| 岗位查询        | 岗位、薪资、招聘、城市       | 岗位需求数据             |
| 能力查询        | 能力项、工作任务、技能要求   | 职业能力分析表           |
| 布点查询        | 布点数、开设院校、层次、年份 | 专业布点数据             |

### 3.3 意图 → 路径 → 结果类型总览

| 用户场景                           | 意图识别    | 检索路径                | 返回结果类型                         |
| ---------------------------------- | ----------- | ----------------------- | ------------------------------------ |
| 什么是直播电商？                   | 内容 / 知识 | 文档检索 + 事实图谱     | 概念定义卡 + 相关事实 + 原文出处     |
| 电商企业融资政策有哪些？           | 内容 / 知识 | 文档检索 + 事实图谱     | 政策措施清单 + 适用对象 + 原文出处   |
| 如何完成数据行列转换？             | 内容 / 任务 | 文档检索 + 任务大纲     | 任务步骤卡 + 前后步骤 + 所需资源     |
| 电子商务专业核心课程有哪些？       | 内容 / 专业 | 文档检索 + 专业卡片     | 专业卡片（课程结构、证书、接续专业） |
| 杭州电商运营岗位薪资分布？         | 数据 / 岗位 | 领域数据查询 + 聚合     | 岗位记录列表 + 薪资分布图            |
| 电商运营岗位需要哪些能力？         | 数据 / 能力 | 领域数据查询 + 树导航   | 能力树（大类 → 任务 → 能力项）       |
| 近三年高职电子商务专业布点数变化？ | 数据 / 布点 | 领域数据查询 + 时序聚合 | 布点透视表 + 时序趋势图              |
| 查看这段政策原文                   | 内容 / 定位 | 原文定位                | 原文页 + 位置高亮                    |

## 四、每类场景返回的结果

平台把检索结果按增强方式统一为 5 类结果卡。

### 4.1 概念 / 政策 / 报告 / 理论类：知识事实卡

**结果结构**：

- 命中片段摘要
- 支撑的事实清单（例如"电商企业 → 可获得 → 债券融资政策"）
- 相关实体（企业、部门、政策名、指标）
- 原文出处（资产名 + 章节 + 页码 + 跳转入口）

**业务价值**：不只返回文本片段，而是把政策 / 理论背后的"谁做什么、发生什么"结构化呈现，便于用户快速抓住要点。

### 4.2 任务 / 操作类：任务步骤卡

**结果结构**：

- 当前步骤内容
- 所属任务名称与背景
- 任务目标 / 要求
- 前一步 / 下一步
- 所需资源（数据文件、模板、工具）
- 期望产物
- 原文出处

**业务价值**：让用户明确"我在做什么、上下步是什么、需要什么资源"，直接支持实操场景。

### 4.3 专业信息类：专业卡片

**结果结构**（一张卡片承载完整专业信息）：

- 专业代码、专业名称
- 修业年限、培养目标
- 职业面向
- 主要专业能力要求
- 专业基础课程 / 核心课程 / 实习实训
- 职业等级证书
- 接续专业（升学路径）
- 原文出处

**业务价值**：一张卡片回答用户 80% 的专业问题，避免拉一堆文本片段。

### 4.4 岗位需求类：岗位数据卡 + 分布视图

**结果结构**：

- 岗位记录列表（岗位类别、城市、学历、薪资、企业性质、企业规模）
- 岗位需求项（技能、素养）
- 聚合视图（按城市 / 学历 / 薪资分档统计）
- 数据集元信息（数据来源、期间、抽取批次）
- 回到源数据集入口

**业务价值**：直接给出画像与分布，支持"哪个城市最多、学历门槛主流是什么、薪资在什么区间"这类问题。

### 4.5 职业能力类：能力树

**结果结构**：

- 分析概要（大类、专业、层次）
- 能力树：大类 → 工作任务 → 工作内容 → 能力项
- 能力项详情（编码、名称、隶属工作内容）
- 数据集元信息

**业务价值**：让能力体系一目了然，支持课程对标、课证融通等判断。

### 4.6 专业布点数类：数据透视

**结果结构**：

- 布点数据透视（专业 × 层次 × 省份 × 年份）
- 时序趋势（近几年变化）
- 地域分布
- 数据集元信息

**业务价值**：支持专业规划、区域布局分析这类决策场景。

### 4.7 所有结果都携带的公共上下文

无论走哪条路径，都会附带：

| 上下文项     | 说明                                                                 |
| ------------ | -------------------------------------------------------------------- |
| 来源资产     | 资产名称、资产类别                                                   |
| 资产版本     | 当前有效版本                                                         |
| 位置信息     | 文档类：章节 / 页码；数据类：源表名 + 行范围                         |
| 跳转入口     | "查看原文" 或 "查看源数据集"                                         |
| 结果类型标签 | 让用户直观看出这是"知识事实卡"、"任务步骤卡"、"专业卡片"还是"数据表" |

## 五、混合场景的处理

用户提问不总是单一意图。例如：

**问题**："数据分析这门课涉及哪些操作技能？配套教材第几章讲？"

平台处理：

1. 意图路由识别到"数据查询（能力）+ 内容查询（任务）"两类信号。
2. 数据路径查职业能力分析表，返回相关能力项。
3. 内容路径查教材，返回任务步骤或章节位置。
4. 结果按主通道分区展示：能力项卡片在上，教材步骤卡片在下。

结论：一次查询允许多类结果并存，前端按类别分区。

## 六、本阶段交付边界

**本阶段覆盖**：

- 概念 / 政策 / 报告等知识型内容查询
- 教材任务操作查询
- 专业简介查询
- 岗位需求 / 职业能力分析 / 专业布点数三类数据查询
- 意图路由（两级） + 双入口 + 5 类结果卡

**本阶段不覆盖（后续版本讨论）**：

- 专业教学标准、人才培养方案、专业人才需求报告、职业技能证书标准的检索增强
- 检索结果的权限与分级过滤（谁能看到哪些资产）
- 检索结果的治理质量过滤（未通过治理 / 待复核的资产是否出现在结果里）
- 检索结果的引用追溯与问答记录固化
- 结构化数据的单元格级高亮跳转

## 七、待业务确认的问题

1. 第二章 7 类典型场景是否覆盖业务用户的主要提问？是否有高频场景需要补充？
2. 混合意图场景（一问同时涉及内容和数据）是否默认双路径并行？还是让用户显式选择？
3. 各类结果卡的默认字段清单，是否需要按不同用户角色（教师 / 教务 / 企业方 / 学生）差异化定制？
4. 数据类结果的常用聚合视图（按城市 / 按年份 / 按学历 / 按层次），业务侧希望预置哪些"常用聚合模板"？
5. 原文定位精度：文档类给"章节 + 页码"，数据类给"源表 + 行范围"，是否满足业务追溯需求？
6. 后续覆盖资产的优先级建议（教学标准 / 人才培养方案 / 人才需求报告 / 证书标准）？
7. 数据类结果是否需要面向业务提供"导出为 Excel / PDF 报告"的能力？
8. 是否需要在结果卡上暴露资产的时效性标签（如"政策 5 年内有效"、"数据截至 2026 年"）？后续与权限治理阶段一并规划。
