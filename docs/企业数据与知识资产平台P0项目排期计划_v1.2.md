# 企业数据与知识资产平台 P0 项目排期计划 v1.2 — NEXUS

## 一、排期基线

本文档基于以下 v2.2 最终版方案，仅覆盖 P0 级别需求，并对 v1.1 排期进行业务专家投入压缩：

- `docs/企业数据与知识资产平台技术选型和架构nexus_v2.2.md`
- `docs/企业数据与知识资产平台需求Spec_v2.2.md`
- `docs/企业数据与知识资产平台Prototype设计文档_v2.2.md`
- `docs/企业数据与知识资产平台P0项目排期计划_v1.1.md`

排期假设：

1. 项目周期按 8 周安排，计划开始时间为 2026-05-06，计划结束时间为 2026-06-30。
2. 每周按 5 个工作日估算，计划工作量计算公式为：计划工作量 = 计划工作日数 * 7.5 小时/人。
3. 后台开发和前端开发按 8 周主力投入估算，每人每周 37.5 小时。
4. 业务专家总投入控制为 2 周，即 10 个工作日 * 7.5 = 75.0 小时/人，不要求每周参与。
5. 投入资源为 1 名业务专家、1 名前端开发、1 名后台开发；后台开发兼任项目负责人和 AI 工程师。
6. 排期表每行按周合并任务，不再拆分为每日或单事项级任务。
7. 每 2 周设置一个具备演示汇报能力的里程碑节点，分别安排在第 2、4、6、8 周。
8. 计划工作量列按角色分别列出，格式为“后台 37.5；前端 37.5；业务专家 X.X”。
9. 本排期未扣除法定节假日、调休和客户现场不可用时间；正式执行前需按企业工作日历校准。
10. 实际开始时间、实际结束时间、实际工作量暂不赋值，待项目执行中更新。
11. 完成进度初始值为 0，后续按实际交付物和评审结果更新。

## 二、P0 交付范围

P0 只交付最小可用闭环：

```text
数据源配置 / 接入
  -> 原始对象留存
  -> 作业编排与解析
  -> 标准化对象
  -> AI 治理建议与质量评分
  -> 规则护栏与治理结果
  -> available / review_required 状态流转
  -> RAGFlow 索引
  -> 权限过滤检索 / QA
  -> 引用追溯与审计
```

P0 不包含：

- 企业 IAM / SSO。
- 自研 `llm-gateway`。
- 独立 `ai-governance-orchestrator`。
- 产品化运维观测中心、发布管理、告警、容量规划。
- D5/D6 全量接入、知识图谱生产化、SFT 语料量产、评价标准库生产化。
- P1/P2 独立产品页面或增强型运营分析能力；如需演示检索/QA，仅提供 P0 最小联调入口。
