## 一、v3.1 设计理念

### 1.1 设计愿景

NEXUS v3.1 控制台定位为**轻量化 SaaS 风格的现代数据资产管理平台**，面向中小型组织（教研团队、院校、中型企业），强调"开箱即用、可视化驱动、零学习曲线"。同时兼顾**企业数据资产管理特有的操作模式**——批量处理、SLA 驱动的审核队列、全链路可追溯、安全合规操作。

### 1.2 七大设计原则（v3.1 修订）

| 原则 | 说明 | 设计表现 |
|------|------|----------|
| **简洁高效** | 一屏一事，核心操作 ≤ 3 步完成 | 减少嵌套弹窗，优先内联编辑和侧边抽屉 |
| **可视化驱动** | 数据状态一目了然，决策有据可依 | 仪表盘、趋势图、状态卡片、交互式血缘 DAG |
| **操作效率优先** | 支持批量处理和快速扫描，减少重复操作 | 批量采纳、批量标签、保存筛选视图、键盘快捷键 |
| **模块化** | 功能独立、按需组合、可扩展 | 组件化设计系统，页面即模块拼装 |
| **云原生美学** | 现代、轻盈、有呼吸感 | 柔和渐变、微阴影、圆角卡片、充足留白 |
| **安全可信** | 高风险操作可防误触，变更可追溯回滚 | 软删除回收站、输入确认、操作预览、变更日志 |
| **响应式友好** | 桌面端优先，兼顾平板 | 弹性栅格、自适应卡片、折叠导航 |

## 三、全局布局框架（v3.1 重构）

### 3.1 导航设计原则

**v3.0 的「收起悬停展开」模式已废弃。** 企业数据资产管理平台中，导航是用户的核心操作锚点，不应隐藏。v3.1 采用**默认展开 + 手动收起**策略：

- 默认宽度 220px 全展开，首次使用即为展开态
- 用户可手动切换至收起态 64px（仅图标），**状态持久化**
- 收起态图标保留待办徽章数字
- 导航项按功能分组，组间有分隔线

### 5.3 交互细节

|------|------|
| 统计卡片点击 | 跳转到对应模块并带入状态筛选 |

```
     · 分级 L3 45% 🔴 → 必须裁定
     · 组织 78% 🟡 → 建议确认
  → 裁定分级为 L4，查看证据 "章节3.2 提及内部决策数据"
  → 确认复核通过
  → Toast：复核通过 + [撤销] (5s)
```
