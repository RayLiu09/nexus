# Task Package: ORC-10 Structured Domain Expansion

## Source Context

- `docs/retrieval_recall_v1_implementation_plan.md`: ORC-10 expands structured retrieval to `job_demand` and `competency_analysis`.
- `nexus_app.retrieval.executors.major_distribution`: reference implementation for guarded structured SQL retrieval.
- `nexus_app.retrieval.domain_registry`: registered domains and query profiles.
- `nexus_app.models`: Pipeline B `job_demand_*` and `occupational_ability_*` domain tables.

## Goal

Add guarded structured retrieval executors for job demand and occupational competency analysis, enabling v1.0 retrieval/recall orchestration to query all three Pipeline B structured pilot domains.

## Scope

- Add `JobDemandRetrievalExecutor`.
- Add `CompetencyRetrievalExecutor`.
- Extend domain registry query profiles and field whitelists.
- Wire executors into `RetrievalOrchestrator`.
- Add tests for guardrails, executor outputs, source refs, and orchestrator dispatch.

## Out Of Scope

- Database migrations.
- Data model changes.
- API or Console contract changes.
- Natural-language-to-SQL execution.
- Permission filtering; v1.0 keeps `access_scope = all_assets`.
- Graph-style retrieval data sources.
- Knowledge Outline work.

## Forbidden Changes

- Do not execute raw SQL generated by LLM.
- Do not allow fields outside registered query profile whitelists.
- Do not introduce external retrieval baselines or graph-style retrieval data sources.
- Do not modify `/open/v1/search`, `/open/v1/qa`, or internal API contracts.
- Do not include unrelated `knowledge_outline` model, migration, or service changes.

## Deliverables

- `nexus-app/nexus_app/retrieval/executors/job_demand.py`
- `nexus-app/nexus_app/retrieval/executors/competency.py`
- `nexus-app/nexus_app/retrieval/domain_registry.py`
- `nexus-app/nexus_app/retrieval/executors/__init__.py`
- `nexus-app/nexus_app/retrieval/orchestrator.py`
- `nexus-app/nexus_app/retrieval/__init__.py`
- `nexus-app/tests/retrieval/test_job_demand_executor.py`
- `nexus-app/tests/retrieval/test_competency_executor.py`
- Focused updates to existing retrieval guardrail/orchestrator tests.

## Acceptance

- Valid `job_demand.record_list`, `job_demand.count_by_city`, `job_demand.count_by_education`, `job_demand.salary_distribution`, and `job_demand.requirement_keyword` plans execute safely.
- Valid `competency.task_tree`, `competency.ability_items_by_category`, `competency.ability_items_by_task`, and `competency.relations_by_ability` plans execute safely.
- Unknown profiles, non-whitelisted filters/group/order fields, non-whitelisted metrics, and over-limit plans fail closed.
- Results use the unified `RetrievalResult` schema and include traceable source refs.
- Orchestrator dispatches structured `job_demand` and `competency_analysis` sub queries to the new executors.
- Tests use SQLite fixtures and do not perform network calls.
