# Task Package: ORC-06 Retrieval Orchestrator Minimal Loop

## Source Context

- `docs/retrieval_recall_v1_implementation_plan.md`: ORC-06 wires intent recognition, retrieval planning, parallel retrieval, and context pack assembly.
- `nexus_app.retrieval.intent`: LiteLLM-backed intent recognition with low-confidence clarification.
- `nexus_app.retrieval.planner`: LiteLLM-backed retrieval plan generation.
- `nexus_app.retrieval.executors`: pgvector unstructured executor and guarded `major_distribution` structured executor.
- `nexus_app.retrieval.schemas`: v1.0 contracts for steps, plans, results, source refs, and context packs.

## Goal

Add the backend orchestration layer that forms the v1.0 minimal retrieval/recall loop:

```text
intent recognition -> query transformation -> retrieval execution -> context pack
```

## Scope

- Add `RetrievalOrchestrator`.
- Add default factory wiring for existing intent, planner, unstructured, and `major_distribution` executors.
- Build conversation steps for retrieval execution and context assembly.
- Support low-confidence clarification, single unstructured retrieval, single structured `major_distribution` retrieval, and mixed plans.
- Add focused unit tests with fake services/executors.

## Out Of Scope

- LLM Markdown summary generation.
- API integration.
- Console integration.
- SSE or realtime transport.
- New structured domains beyond `major_distribution`.
- Permission filtering; v1.0 keeps `access_scope = all_assets`.
- Database migrations.

## Forbidden Changes

- Do not call LiteLLM directly from the orchestrator; use injected/default services.
- Do not call embedding providers directly.
- Do not execute raw SQL generated by LLM.
- Do not introduce RAGFlow or Evidence Graph as retrieval sources.
- Do not persist user query text, answer text, Prompt text, source content, or API keys.
- Do not change `/open/v1/search` or `/open/v1/qa` contracts.

## Deliverables

- `nexus-app/nexus_app/retrieval/orchestrator.py`
- `nexus-app/tests/retrieval/test_retrieval_orchestrator.py`
- Export updates in `nexus-app/nexus_app/retrieval/__init__.py`.

## Acceptance

- Low-confidence intent returns `needs_clarification` and does not call planner or executors.
- High-confidence unstructured query returns a completed context pack with plan, result, steps, and source refs.
- High-confidence `major_distribution` structured query returns a completed context pack with structured result and source refs.
- Mixed plans execute multiple sub queries and merge results/source refs.
- Executor failure is captured as a failed sub-query result; available results still form a partial context pack.
- Tests use fakes and do not perform network calls.
